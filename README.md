# 🚀 AWS Infrastructure Automation using Terraform

## 📌 Project Overview
This project demonstrates how to provision and manage AWS infrastructure using **Terraform (Infrastructure as Code)**. It follows a structured approach to automate cloud resource creation, ensuring scalability, consistency, and efficiency.

The goal of this project is to showcase practical DevOps skills by deploying cloud infrastructure in an automated and reusable way.

---

## 🎯 Objectives
- Automate AWS resource provisioning using Terraform
- Implement Infrastructure as Code (IaC) best practices
- Reduce manual configuration and human errors
- Create reusable and scalable infrastructure components
- Demonstrate real-world DevOps workflow

## ⚙️ How It Works

1. **Provider Configuration**
   - Defines AWS as the cloud provider
   - Configures region and authentication

2. **Variables**
   - Inputs like region, instance type, etc.
   - Makes project reusable and flexible

3. **Main Configuration**
   - Defines AWS resources (e.g., EC2, S3, etc.)
   - Infrastructure is declared in code

4. **Outputs**
   - Displays useful information after deployment
   - Example: public IP, resource IDs

## 🚀 Setup & Execution

### Step 1: Clone Repository

git clone https://github.com/paladugubharath/Aws-Infrastructure-Automation-using-Terraform.git

cd Aws-Intrastrucure-Automation-using-Terraform

### Step 2: Initialize Terraform

terraform init

### Step 3: Plan Infrastructure

terraform plan

### Step 4: Apply Configuration

terraform apply

### Step 5: Destroy Infrastructure (Optional)

terraform destroy
